# Примерный план работ
- Написать функции для модуля KudaGo:
  - get_events - список событий (наименование, дата, описание)
  - get_events_details - детализация события (если нет описания в списке событий, взять тут, если есть картинка тоже вззять тут)
  - get_collections - подборки - проверить, возможно там есть что-то интересное для ежедневных уведомлений
  - get_places - спсиок мест - тоже проверить, возмоожно будет полезно, например если в списке событий и в детализации нет адреса, то будем брать отсюда
  - get_movie_list - список фильмов
  - get_show_list - спсиок показов, оба запроса список фильмов и список показов относятся к кино, нужно будет разобраться оставлять обе функции или достаточно будлет одной
  - collect_data_for_sending - написать функцию, которая будет использовать все выше инструкции для сбора данных и формирования их в единую структуру для отправки
- Написать функции для модуля взаимодействия с ТГ:
  - send_message - функция для отправки сообщения
  - set_webhook - связать программу с ботом
- Написать функции для модуля main:
  - index - ручка/эндпоинт для с http методом POST, который выполняет функции из других модулей и отправляет данные на сервер ТГ
  - ? - подумать и реализовать логику отправки сообщений в бот один раз в сутки.
    - Из вариантов использовать бесконечный цикл с timeout в 24 часа
    - Использовать поток в фоне с тем же timeout в 24 часа
    - Погуглить, возможно есть варианты проще.
    - **ВАЖНО**: Так же это должно работать для каждого, кто подключает бота, а не один раз. Возможно нужно будет подключить встроенную базу данных в которых будут храниться иденнтификаторы пользователя и время старта бота, чтобы от этого времени можно было бы оттталкиваться
  - to_unixtime - скорей всего потребуется вспомогательная функция для перевода datetime "2025-19-11 15:58:00" в unixtime 1763546329
  - main - для запуска сервера и приложения
- Телеграм
  - В ТГ боте @BotFather
    - Создать бота
    - Получить токен
    - Настроить кнопку /start
# Структура проекта на сегодняшний день
```bash
➜  EventNotify git:(main) tree -a -L1                                                                                                                                                           11/19/25 16:09
.
├── app
├── .git
├── .gitignore
├── pyproject.toml
├── .python-version
├── README.md
└── reports
```
```bash
➜  EventNotify git:(main) tree                                                                                                                                                                  11/19/25 16:10
.
├── app
│   ├── api_event
│   ├── api_telegram
│   └── main.py
├── pyproject.toml
├── README.md
└── reports
    └── day_001.md
```
- Директория `app` содержит в себе два пакета `api_event` и `api_telegram` и модуль `main.py` - в пакетах будут модули, в которых выше опсанная логика взаимодействия с апи telegram и kuda go, в модуле `main.py` запуск программы.
- Файл `pyproject.toml` содержит краткое описание проекта (версия языка, имя проекта, версия проекта) и зависимостей (устаноавливаемые библиотеки, например библиотека реализующая сервер)
- Файл `README.md` будет содержать полное описание проекта и как его запустить
- Директоряи `reports` содержит `.md` файлы с текстовой историей о том как продвигается написание программы.
- Файл `.git` нужен для сохранения истории изменения в проекте, так же помгает // писать несколько версий проекта в одном репозитории, не пересекая кодовую базу. Или наоборот, пока основная кодовая база работает, можно создать вторую ветку кодовой базы и писать новый/дополнительный функционал не повредив при этом основной проект. А после внедрения новго функционадла и его тестирования, ветки можно объединить.
- Файл `.gitignore` содержит шаблоны/маски того что не нужно загружать в репозиторий. Например чувствительные данные: токены, пароли, временные файлы которые появляются в процессе запуска и работы программы и т.д.
- Файл `.python-version` - содержит только версию python, нужен для менеджера зависимостей.
